<UUID란>
- Universally Unique Identifier 범용 고유 식별자의 약자로 네트워크 상에서 고유성이 보장되는 아주 긴 랜덤한 식별자를 생성하는 표준 방식.

[언제 사용될까?]
기본 키(Primary Key): 데이터베이스 레이블의 각 행을 고유하게 식별하는 ID로 사용할 때.

[장점]
- 예측 불가능: id=1, id=2처럼 순차적이지 않아서 외부에서 다음 ID를 예측하거나 추측하기 어려워 보안상 이점이 있습니다.
- 분산 시스템: 여러 데이터베이스 서버나 시스템에서 동시에 데이터를 생설할 때 ID 충돌 없이 고유성을 보장하기 쉽습니다.
- 외부 노출: API 응답 등에서 내부 데이터 순서를 노출하지 않고 고유한 ID를 제공할 수 있습니다.

[단점]
- 정수 ID보다 저장 공간을 더 많이 차지하고, 정수 ID보다 인덱싱 성능이 약간 느릴 수 있습니다.

[Task 모델에서의 사용]
id = models.UUIDField(primary_key=True, default=uuid.uuid4, editable=False)
models.UUIDField:UUID값을 저장하는 필드 타입
primary_key=True: 이 필드를 테이블의 기본 키로 사용합니다.
default=uuid.uuid4:새로운 Task객체가 생성될 때, uuid 모듈의 uuid4()함수를 호출하여 랜덤한 UUID값을 자동으로 생성하여 기본값으로 설정
editable=False: Django Admin등에서 이 필드를 직접 수정할 수 없도록 합니다.
